
# Лабораторная работа №5 - Управление качеством и зрелостью процессов проектирования информационных систем (CMMI, ISO/IEC 15504 SPICE, ITIL Service Management)
 

## 1. Цель работы

Изучить современные модели зрелости процессов (CMMI, SPICE) и лучшие практики управления качеством в ИС, применяемые в корпоративной и промышленной разработке. Научиться проводить базовую оценку зрелости процессов, анализировать GAP-разрывы и формировать предложения по их улучшению.

---

## 2. Теория

### 2.2. Модель CMMI (Capability Maturity Model Integration)

CMMI — одна из наиболее признанных мировых моделей зрелости.
Она определяет **5 уровней зрелости**:

| Уровень | Название               | Характеристика                                   |
| ------- | ---------------------- | ------------------------------------------------ |
| 1       | Initial                | Процессы хаотичны, результаты непредсказуемы     |
| 2       | Managed                | Процессы управляются на уровне проектов          |
| 3       | Defined                | Процессы стандартизированы на уровне организации |
| 4       | Quantitatively Managed | Процессы измеряются, используются метрики        |
| 5       | Optimizing             | Постоянное улучшение, аналитика, инновации       |

Области применения CMMI: системная инженерия, программная разработка, управление услугами.

---

### 2.3. Модель SPICE / ISO/IEC 15504

SPICE оценивает **способности** процессов по диапазону уровней:

| Уровень | Название    | Суть                             |
| ------- | ----------- | -------------------------------- |
| 0       | Incomplete  | Процесс не выполняется           |
| 1       | Performed   | Процесс выполняется              |
| 2       | Managed     | Управляется и планируется        |
| 3       | Established | Документирован, стандартизирован |
| 4       | Predictable | Контролируемый, измеримый        |
| 5       | Optimizing  | Постоянно улучшается             |

SPICE ориентирован на:

* аналитическую оценку процессов;
* независимый аудит;
* оценку поставщиков;
* сертификацию подразделений.

---

### 2.4. ITIL: управление качеством ИТ-услуг

ITIL — библиотека практик по управлению ИТ-услугами.
Основные процессы:

* Incident Management — управление инцидентами;
* Change Management — управление изменениями;
* Problem Management — анализ инцидентов и предотвращение повторений;
* Service Level Management — SLA, KPI;
* Release & Deployment Management — управление релизами;
* Continual Service Improvement (CSI) — непрерывное улучшение.

Роль ITIL в зрелости:

* обеспечивает стабильность процессов,
* повышает качество обслуживания,
* улучшает предсказуемость работы сервисов.

## 3. Практическое задание

### Часть 1. Мини-диагностика зрелости процесса проектирования ИС

Выберите учебный или реальный процесс разработки (например, анализ требований, QA, CI/CD, управление изменениями).

Выполните:

1. Опишите процесс (1–2 абзаца).
2. Оцените его уровень зрелости **по CMMI (1–5)**.
3. Оцените его уровень зрелости **по SPICE (0–5)**.
4. Обоснуйте выбранный уровень зрелости.
5. Заполните таблицу GAP-анализа:

| Область | Текущий уровень | Требуемый уровень | Разрыв (GAP) | Рекомендации |
| ------- | --------------- | ----------------- | ------------ | ------------ |

---

### Часть 2. Краткая ITIL-классификация выбранного процесса

Определите:

1. Какие ITIL-процессы связаны с вашим выбранным процессом?
2. Какие артефакты и документы должны существовать?
3. Какие метрики (KPI/SLA) подходят?
4. Какие риски возникают при низкой зрелости?

---

### Часть 3. Предложение по улучшению зрелости

На основе анализа:

1. Определите 3–5 улучшений, которые могут повысить зрелость процесса.
2. Постройте цепочку улучшений в стиле ITIL CSI:

**Assess → Plan → Implement → Review**

3. Сделайте краткий вывод (5–7 предложений).

---

## 4. Контрольные вопросы

1. В чем принципиальное отличие уровней зрелости CMMI 2 и 3?
2. Почему уровень зрелости SPICE 4 требует наличия метрик и предсказуемости?
3. Как ITIL помогает устранить проблемы, возникающие при низкой зрелости процессов?
4. Какие риски для бизнеса возникают при наличии уровня зрелости CMMI = 1?
5. Какие группы процессов ИТ-услуг наиболее критичны для разработки и эксплуатации ИС?
